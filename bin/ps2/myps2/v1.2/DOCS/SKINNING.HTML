Introduction

A skin for myPS2 consists of a bunch of XML and image files. For every menu in
myPS2 there is an XML file which defines all controls of the menu, where they
appear on the screen, how they look, what images they use etc.

The whole layout of the myPS2 skinning system is very similar to the one in XBMC,
if you have ever made a skin for that you can probably start right away.

To make sure the syntax of your XML files is correct you can load them in Firefox
or Internet Explorer. If they are alright they will show up in a tree structure. If you
have made some mistake you will get an error message telling you what's wrong
and in which line the problem is.

You can use JPG and PNG images in your skin. PNG images can optionally be given
an alpha channel.

All of your skins go into the subdirectory /skins/ in the myPS2 directory. The directory
structure of a skin should look like this:

    /myPS2/skins/mySkin/                - Skin base directory for 'mySkin' skin
    /myPS2/skins/mySkin/media/       - All of your images go into this directory
    /myPS2/skins/mySkin/pal/           - All of your XML files for PAL go here
    /myPS2/skins/mySkin/ntsc/         - All of your XML files for NTSC go here
    /myPS2/skins/mySkin/fonts/        - All of the fonts your skin uses go here

The skin base directory must contain a file named skin.xml which holds some general
information about the skin. The format of the XML file is as follows:

    <skin>
        <name>myPS2 default skin</name>
        <default>pal</default>
    </skin>

Name is the name that appears in the skin selection menu in myPS2.
default is the default loading directory of the skin (read below).

A skin can have 2 different sets of XML files, one for PAL and one for NTSC. This is
because PAL and NTSC run at different display resolutions and some controls may
need to be finetuned for each display mode.

Under PAL the screen resolution is 640 * 512 where as under NTSC it is 640 * 448.

myPS2 will try to load the appropriate set of files depending on your selected display
mode, so if you are running myPS2 on a NTSC console it will look for an NTSC sub
directory in the skin directory first and try to load the files from there.
If myPS2 can not find an XML file in the NTSC directory it will then look for this file
in the directory specified under the <default> </default> field in the skin.xml file.
This way you only have to duplicate the XML files that are actually different for
PAL and NTSC mode.

A menu skin file looks like this :

    <window>
        <defaultcontrol>100</defaultcontrol>
        <controls>
                 controls are defined here..
        </controls>
    </window>

defaultcontrol is the id of the control to receive the focus when the menu is opened
for the first time. All controls are defined within the <controls> </controls> section.

The best way to make a new skin is to make a copy of the existing myPS2 skin and
start making modifications to that copy.

Control Reference

This section contains information about the myPS2 controls.
- Label
    a static control for displaying lines of text

- Image
    a static control for displaying an image

- Button
    a clickable button that can either be just a text label, an image or a combination
    of both.

- Thumbnailpanel
    a control that displays a panel of thumbnails. Used in the Picture Viewer menu

- List
    a control for displaying a list of items

- Dirview
    a control for displaying a directory structure

- Progressbar
    a progress bar that indicates the status of a pending operation

 - Slider
    a movable slider control

- Combobutton
    a button that can have several selections

- Togglebutton
    a button that can have 2 states, on or off



Label Control
        <control>
          <description>My Pictures Label</description>
          <type>label</type>
          <id>194</id>
           <posX>120</posX>
          <posY>60</posY>
          <width>200</width>
          <align>center</align>
          <label>1</label>
          <textcolor>0x80FFFFFF</textcolor>
          <font>arial_10</font>
         </control>

    <description> </description>
        This field is ignored by the parser and is just a comment to help the skinner

    <type> </type>
        This specifies the control type. Must be label for a label control

     <id> </id>
        This is used to identify the control from the source code. ID 102 could be
        the identifier for the "Sort By Name" button in the "My Pictures" menu,
        for example. The source code will then know what to do when the control
        is receives user input.
        Not all controls need an id field, only those that are directly referenced
        from the source code. Look at the default myPS2 skin to see which menu
        ids are required for every menu.

    <posX> </posX>
        This specifies the horizontal position of the control on the screen. The value
        should be between 0 (left side of the screen) and 640 (right side of the screen).
       
    <posY> </posY>
        This specifies the vertical position of the control on the screen. The value
        should be between 0 (top of the screen) and 512 for PAL or 448 for NTSC,
        respectively.

    <width> </width>
        This specifies the width of the control. If the text is too long to fit into the
        control it will be automatically shortened to make sure it doesn't extend this
        width. If no width is specified it will be calculated by myPS2 dynamically.

    <align> </align>
        Specifies the alignment of the text string within the label control. Can be
        left to align the text to the left side of the control, right or center.
        The text is centered by default.

    <label> </label>
        This is the ID of a string entry from the language xml file to be displayed
        by the label control.

    <textcolor> </textcolor>
        This specifies the color the text will be drawn in. The format is ARGB
        (Alpha Component, Red Component, Green Component, Blue Component).
        The Alpha value must be between 0x00 and 0x80 where 0x00 is total
        transparency and 0x80 total opaqueness. The other 3 values
        must be between 0x00 and 0xFF.

    <font> </font>
        Name of a font from the font xml file.





Image Control
        <control>
          <description>Some Image</description>
          <type>image</type>
          <id>182</id>
          <posX>50</posX>
          <posY>140</posY>
          <width>200</width>
          <height>200</height>
          <texture>select.png</texture>
          <colordiffuse>0x80FF0000</colordiffuse>
          <keepaspectratio>1</keepaspectratio>
        </control>

    <description> </description>
        This field is ignored by the parser and is just a comment to help the skinner

    <type> </type>
        This specifies the control type. Must be image for a image control

     <id> </id>
        This is used to identify the control from the source code. ID 102 could be
        the identifier for the "Sort By Name" button in the "My Pictures" menu,
        for example. The source code will then know what to do when the control
        is receives user input.
        Not all controls need an id field, only those that are directly referenced
        from the source code. Look at the default myPS2 skin to see which menu
        ids are required for every menu.

    <posX> </posX>
        This specifies the horizontal position of the control on the screen. The value
        should be between 0 (left side of the screen) and 640 (right side of the screen).
       
    <posY> </posY>
        This specifies the vertical position of the control on the screen. The value
        should be between 0 (top side of the screen) and 512 for PAL or 448 for NTSC,
        respectively.

    <width> </width>
        This specifies the width of the control. The used texture will be resized to this.

    <height> </height>
        This specifies the height of the control. The used texture will be resized to this.

    <keepaspectratio> </keepaspectratio>
        If this is 1 the image will be resized to the specified dimensions but the original
        aspect ratio of the image is maintained. This is 0 by default.

    <colordiffuse> </colordiffuse>
        This specifies the diffuse color which can be used to affect the final output of
        the texture color. The format is ARGB.
        The Alpha component must be between 0x00 and 0x80. The other components
        must be between 0x00 and 0xFF.

    <texture> </texture>
        This specifies the texture to use for the image control. It must be located in the
        /media/ subdirectory of your skin and can be either a JPG or PNG file.





Button Control
        <control>
          <description>Some Button</description>
          <type>button</type>
          <id>102</id>
          <posX>50</posX>
          <posY>140</posY>
          <width>200</width>
          <height>200</height>
          <texturefocus>button_0.png</texturefocus>
          <texturenofocus>button_1.png</texturenofocus>
          <texturewidth>200</texturewidth>
          <textureheight>200</textureheight>
          <texturealign>center</texturealign>
          <colordiffuse>0x80FF0000</colordiffuse>
          <label>99</label>
          <textcolor>0x80FFFFFF</textcolor>
          <selectedcolor>0x8000FF00</selectedcolor>
          <disabledcolor>0x80FF0000</disabledcolor>
          <font>arial_10</font>
          <align>left</align>
          <aligny>top</aligny>
          <textoffsetx>2</textoffsetx>
          <textoffsety>4</textoffsety>
          <onup>101</onup>
          <ondown>103</ondown>
          <onleft>120</onleft>
          <onright>140</onright>
          <visible>0</visible>
        </control>

    <description> </description>
        This field is ignored by the parser and is just a comment to help the skinner

    <type> </type>
        This specifies the control type. Must be button for a button control

    <id> </id>
        This is used to identify the control from the source code. ID 102 could be
        the identifier for the "Sort By Name" button in the "My Pictures" menu,
        for example. The source code will then know what to do when the control
        is receives user input.
        Not all controls need an id field, only those that are directly referenced
        from the source code. Look at the default myPS2 skin to see which menu
        ids are required for every menu.

    <posX> </posX>
        This specifies the horizontal position of the control on the screen. The value
        should be between 0 (left side of the screen) and 640 (right side of the screen).
       
    <posY> </posY>
         This specifies the vertical position of the control on the screen. The value
        should be between 0 (top side of the screen) and 512 for PAL or 448 for NTSC,
        respectively.

    <width> </width>
        This specifies the width of the control. The used texture will be resized to this.

    <height> </height>
        This specifies the height of the control. The used texture will be resized to this.

    <texturefocus> </texturefocus>
        Image displayed while button is selected

    <texturenofocus> </texturenofocus>
        Image displayed when button is not selected

    <texturewidth> </texturewidth>
        If specified texture will be resized to this. Otherwise texture will be resized to the width
        of the button control.

    <textureheight> </textureheight>
        If specified texture will be resized to this. Otherwise texture will be resized to the height
        of the button control.

    <texturealign> </texturealign>
        This is only important if they texture dimensions are different to the dimensions of the
        control. Specifies the alignment of the texture within the button control. Can be left,
        center or right (center by default)

    <colordiffuse> </colordiffuse>
        This specifies the diffuse color which can be used to affect the final output of
        the texture color. The format is ARGB.
        The Alpha component must be between 0x00 and 0x80. The other components
        must be between 0x00 and 0xFF.
   
    <label> </label>
         This is the ID of a string entry from the language xml file. The string will be
        displayed in the button control if this is specified.

    <textcolor> </textcolor>
        Color of the text if the button has a text label. Format is ARGB.

    <selectedcolor> </selectedcolor>
        Color of the text when the button has the focus. Format is ARGB.

    <disabledcolor> </disabledcolor>
        Color of the text if the button control is disabled.

    <font> </font>
        Name of a font from the font xml file, used for the optional text label.

    <align> </align>
        Specifies the alignment of the text label in the button control. Can be left, center
        or right.

    <aligny> </aligny>
        Specifies the vertical alignment of the text label in the button control. Can be top
        or center.

    <textoffsetx> </textoffsetx>
        Amount of pixels to offset text label from left/right edge when using align left or right.

    <textoffsety> </textoffsety>
        Amount to pixels to offset text label from top when using aligny top.

    <onup> </onup>
        ID of the control to move to when the gamepad up button is pressed.

    <ondown> </ondown>
        ID of the control to move to when the gamepad down button is pressed.

    <onleft> </onleft>
        ID of the control to move to when the gamepad left button is pressed.

    <onright> </onright>
        ID of the control to move to when the gamepad right button is pressed.

    <visible> </visible>
        If this is specified and set to 0 control will be invisible. All controls are visible by
        default.



Thumbnailpanel Control
        <control>
          <description>A thumbnail panel</description>
          <type>thumbnailpanel</type>
          <id>108</id>
          <posX>50</posX>
          <posY>40</posY>
          <width>400</width>
          <height>400</height>
          <itemtexture>item.png</itemtexture>
          <itemtexturefocus>itemfocus.png</itemtexturefocus>
          <itemwidth>100</itemwidth>
          <itemheight>100</itemheight>
          <texturewidth>80</texturewidth>
          <textureheight>80</textureheight>
          <paddingx>20</paddingx>
          <paddingy>20</paddingy>
          <thumbwidth>64</thumbwidth>
          <thumbheight>64</thumbheight>
          <thumbalign>center</thumbalign>
          <thumboffsetx>0</thumboffsetx>
          <thumboffsety>0</thumboffsety>
          <textcolor>0x80FFFFFF</textcolor>
          <selectedcolor>0x8000FF00</selectedcolor>
          <font>arial_10</font>
          <texturedown>scrolldown.png</texturedown>
          <textureup>scrollup.png</textureup>
          <scrolldownposx>380</scrolldownposx>
          <scrolldownposy>380</scrolldownposy>
          <scrollupposx>360</scrollupposx>
          <scrollupposy>380</scrollupposy>
          <onup>101</onup>
          <ondown>103</ondown>
          <onleft>120</onleft>
          <onright>140</onright>
          <visible>0</visible>
        </control>

    <description> </description>
        This field is ignored by the parser and is just a comment to help the skinner

    <type> </type>
        This specifies the control type. Must be thumbnailpanel for a thumbnail panel
        control

     <id> </id>
        This is used to identify the control from the source code. ID 102 could be
        the identifier for the "Sort By Name" button in the "My Pictures" menu,
        for example. The source code will then know what to do when the control
        is receives user input.
        Not all controls need an id field, only those that are directly referenced
        from the source code. Look at the default myPS2 skin to see which menu
        ids are required for every menu.

    <posX> </posX>
        This specifies the horizontal position of the control on the screen. The value
        should be between 0 (left side of the screen) and 640 (right side of the screen).
       
    <posY> </posY>
         This specifies the vertical position of the control on the screen. The value
        should be between 0 (top side of the screen) and 512 for PAL or 448 for NTSC,
        respectively.

    <width> </width>
        This specifies the width of the control. This determines how many items
        fit in a row.

    <height> </height>
        This specifies the height of the control. The determins how many rows
        of items fit on the screen.

    <itemtexture> </itemtexture>
        This specifies the texture of an item in the panel.

    <itemtexturefocus> </itemtexturefocus>
        This specifies the texture of an item in the panel that has the focus.

    <itemwidth> </itemwidth>
        This specifies the width of an item in the panel. This determines how many items
        fit in a row.

    <itemheight> </itemheight>
        This specifies the height of an item in the panel. This determines how many rows
        of items fit on the screen.

    <texturewidth> </texturewidth>
        This specifies the width of the item texture. Obviously it should not be bigger than
        the item's width.

    <textureheight> </textureheight>
        This specifies the height of the item texture. Obviously it should not be bigger than
        the item's height.

    <paddingx> </paddingx>
        Amount of pixels to leave free between two items in the panel.

    <paddingy> </paddingy>
        Amount of pixels to leave free between two rows of items in the panel.

    <thumbwidth> </thumbwidth>
        Width of the thumbnail image displayed inside an item container. It must not be bigger
        than itemwidth. Note that the thumbnail will maintain its aspect ratio when it is resized
        to the specified width.

    <thumbheight> </thumbheight>
        Height of the thumbnail image displayed inside an item container. It must not be bigger
        than itemheight. Note that the thumbnail will maintain its aspect ratio when it is resized
        to the specified height.

    <thumbalign> </thumbalign>
        This specifies the alignment of the thumbnail image inside the item container.
        It can be center or left. It is centered by default.

    <thumboffsetx> </thumboffsetx>
        Amount of pixels to offset the thumbnail from the left side of the item container if
        thumbalign is left. Unimportant if the thumbnail is centered.

    <thumboffsety> </thumboffsety>
        Amount of pixels to offset the thumbnail from the top of the item container.

    <textcolor> </textcolor>
        Color of the text displayed beneath a thumbnail. Format is ARGB. The text label
        is vertically aligned with the bottom of the itemtexture and is center aligned
        horizontally.

    <selectedcolor> </selectedcolor>
        Color of the text displayed beneath a thumbnail when it has focus. Format is ARGB.
        The text label is vertically aligned with the bottom of the itemtexture and is center
        aligned horizontally.

    <font> </font>
        Name of a font from the font xml file.

    <texturedown> </texturedown>
        Texture of the scroll-down button if the thumbnail panel has several display pages.
        This will only show if the panel can be scrolled down.

    <textureup> </textureup>
        Texture of the scroll-up button if the thumbnail has several display pages.
        This will only show if the panel can be scrolled up.

    <scrolldownposx> </scrolldownposx>
        Specifies the horizontal position of the scroll-down button. This is relative to the
        upper left corner of the thumbnail panel.

    <scrolldownposy> </scrolldownposy>
        Specifies the vertical position of the scroll-down button. This is relative to the
        upper left corner of the thumbnail panel.

    <scrollupposx> </scrollupposx>
        Specifies the horizontal position of the scroll-up button. This is relative to the
        upper left corner of the thumbnail panel.

    <scrollupposy> </scrollupposy>
        Specifies the vertical position of the scroll-up button. This is relative to the
        upper left corner of the thumbnail panel.

    <onup> </onup>
        ID of the control to move to when the gamepad up button is pressed.

    <ondown> </ondown>
        ID of the control to move to when the gamepad down button is pressed.

    <onleft> </onleft>
        ID of the control to move to when the gamepad left button is pressed.

    <onright> </onright>
        ID of the control to move to when the gamepad right button is pressed.

    <visible> </visible>
        If this is specified and set to 0 control will be invisible. All controls are visible by
        default.

Font Files

In the current version of myPS2 only bitmap fonts are supported. A myPS2 font consists of two
files : a PNG image and a DAT file.

The PNG image contains all the characters and can be generated with a bitmap font tool. The DAT
file contains the width of each character.

You can use any bitmap font tool that can export characters widths. The recommended program is
Bitmap Font Builder, and it will be used in this example:

    1.    Create the font image. Open Bitmap Font Builder and select the font face, size etc. Make
           sure you have selected "Full ASCII Character Set" under "Character Set". You should
           also enable "Font Smoothing" as it makes the font look alot nicer. Now click the button
           next to the "Underline" button and select "Left" as the characters must be left aligned.

           You can now save the files. Click "File" -> "Save as 32-bit TGA (RGBA)" and save the TGA
           file. Then click "File" -> "Save Font Widths (Byte format)" and save the DAT file as well.

           Unfortunately Bitmap Font Builder can not save the font image directly as PNG so you will
           have to open the TGA file you just created with your favourite image program and re-save
           it as PNG yourself.

    2.    Copy your 2 files in the fonts directory.

           Each skin can have a fonts directory in its base directory. When myPS2 tries to load a font
           from the font.xml file (read below) it will first look for the files in the "fonts" directory of the
           selected skin. If it can not find the font files in this directory it will then look for them in the
           "fonts" directory in the myPS2 root directory.

    3.    Create an entry in the font.xml file of your skin for your new font.

           The font.xml file defines all fonts that your skin uses. An entry in the file looks like this :

            <font>
                <name>arial_10</name>
                <file>arial_10.png</file>
            </font>

            name is the name of the font entry.
            file is the filename of the PNG file. You don't have to specify the name of the DAT file as
            myPS2 expects it to be the same as the PNG name, i.e. the DAT file in this example would
            have to be named arial_10.dat.

            The font.xml file can have several fontsets. A fontset specifies which font files should be
            used for a certain character encoding. The default character encoding myPS2 uses
            is Latin-1 (also known as ISO-8859-1). Through fontsets you can supply font files
            for other character encodings to support languages like Greek, Polish, Russian etc.
            Look at the font.xml file of the default myPS2 skin to see how it works.


Language File

The language file holds all strings that are used by the menus in myPS2. The default myPS2
language file is located under /myPS2/language/english.xml

An entry in the file looks like this:

        <string>
            <id>3</id>
            <value>My Files</value>
        </string>

The id field is a unique number used to identify the string from the source code. You don't have
to care about it when translating the file. Just make a copy of the english.xml file, rename it and
translate all of the strings entries.

At the moment only ASCII characters 32-126 can be used!
http://www.lookuptables.com/

Copy your translated file into /myPS2/language/ and it will show up in the language selection
menu in myPS2 where it can then be selected.




Control IDs

Control IDs are used to associate menu controls with certain actions. Some menus expect certain
controls to be present, for instance the "My Pictures" menu looks for a control with the ID 108 and
expects this control to be a Thumbnail Panel. Menu specific IDs start at 100. To learn which IDs
each menu requires have a look at the default myPS2 skin.

There are also some special IDs (0-99) that can be used in any menu. They can be used to display
information such as the name of the MP3 file that is currently being played or the current time. This
list gives you a short overview:

ID Control Type Description
2 Label Playtime of the MP3 / Stream that is currently being played.
3 Label Length of the MP3 that is currently being played.
4 Label Name of the MP3 that is currently being played.
5 Image Images with this ID will only be visible when an MP3 is playing in the background. Otherwise they are invisible.
6 Label Date in format Weekday, Month, Day
7 Label Weekday as string
8 Label Weekday as number (01-31)
9 Label Month as string
10 Label Month as number (01-12)
11 Label Year in format YYYY
12 Label Current time in format HH:MM
